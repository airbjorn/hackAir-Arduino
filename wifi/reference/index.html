Advanced Commands

The WiFi shield communicates with the Arduino using a serial (Tx/Rx, 5V) connection. This page contains all the information regarding the communication protocol to develop custom libraries or use the shield without an Arduino.

General

The shield SoC (ESP8266) runs the NodeMCU firmware with the hackAir routine (init.lua). The routine will listen on the serial port for any incoming commands. In the meantime it will handle WiFi connectivity, if a configuration is stored (SSID & password) Internet connection will be established. Otherwise an AP will be created so the user can enter their WiFi’s credentials.

Commands

Commands begin with e+ so the serial pins can be used with other devices or USB communications. Command parameters, when needed, are comma separated.

  • e+raw
    Switches to Lua interpreter mode. To switch back to command mode one can restart the SoC with node.restart()

  • e+restart
    Reboots the ESP8266.

  • e+version
    Returns the version number as a single integer.

  • e+token
    Sets the authentication token for sending data to the hackAir servers. Example: e+token 123testtoken

  • e+send
    Sends data to the hackAir servers and returns the HTTP status code. If the WiFi is not connected it does nothing. The command takes 5 parameters, in this order: PM2.5 counts, PM10 counts, battery level, tamper indicator, error number. Example: e+send 2.5,10,0,0,0

  • e+clearap
    Clears the stored WiFi configuration and restarts the SoC

  • e+isready Returns 1 if the WiFi connection is established.

  • e+wifi Sets the SSID and password for the WiFi connection. SSIDs with commas or spaces are currently not supported. Example: e+wifi MyPoint,Password
